Multiple Representations

Multiple Representations

Description

How to find it?

Clustering and merging usually are used to fix the classify and merge same records which has been identified into different data improperly. In this dataset, this mainly happened in the attribute "Project Name". So, using the text facet to analysis the situation of this attribute.

1. Same name of department are divided into different groups.

2. Different representation of same department. like Department of Agriculture, abbreviation DOA, Agriculture Department.

How to deal with it?

1. Using the "cluster" to analysis and identify which items are same, then click them and merge them under the same "Project Name".

2. Change different representation into the same one.

Data Cleaning
Inconsistent Formats

Inconsistent Formats

Description

How to find it?

1. Extract Symbol Format: The data in the column of "Lifecycle Cost" has some number with dollar sign "($m)". It will cause some problem in visualisaion. Using Numberic Facet and choose Non-numberic can find these wrong data.

2. Number Format: Some numbers have extra comma separating like '1,000'.

How to deal with it?

1. Using Edit Cells-->Transform, type in the Expression value.replace('($m)',"") to remove all doller sign. Then use Edit Cells --> Common Transforms--> To number to rule the format.

2. Using Numberic Facet on "Project ID", then choose the Non-numeric to change it by using Expression value.replace(",","") to remove all "," in Project ID.

Data Visualisation1

Please click the bar below to see the detail.

The audience of this visualisation

People who want to know the Cost, Schedule and Risk status of projects from all departments of US government.

How this visualisation satified these audiences

This visualisation includes five charts. Each chart can interact with other charts to filter any information you want. Audiences could use different combination of filters to answear some specific questions like "How much money department of Argriculture spend on high risk relyed projects?".

1. Cost variance chart, which to show the percentage of Profit, Loss, Equal and Unknown. This result based on the cost variance data which is the differece between the planned cost and the real cost.

2. Schedule variance chart, which to show the percentage of Punctual, Delay and Unknown. This result based on the schedule variance data which is the difference between the planned completion date and the real completion date.

3. Risk status chart, which to show the percentage of high risk, medium risk and low risk projects. This result based on the cost variance and schedule variance.

4. Agency Cost chart, which to show the cost of each agency. This result based on the sum of the cost of all project in same department.

5. Table chart, which contains the detail related information about all filtered projects.

Data Visualisation2

The audience of this visualisation

People who want to know the Cost and duration of projects in each department yearly.

How this visualisation satified these audiences

This visualisation includes four charts. Each chart can interact with other charts to filter any information you want.

1. Year Cost chart, which to show the cost in each year. This result based on the actual cost data and start date of each project.

2. Duration chart, which to show the duration time of project every year. Using the Agency duration chat and this chart, audiences could easily see the project duration time of each department each year. This result based on the "duration time" which is manipulated into the dataset by me, which is the difference between the "Actual Completion Date" and "Start Date".

3. Agency duration chart, which to show the sum of duration time in each department. This result based on the "Actual Cost" and "Agency name".

4. Table chart, which to show the detail of these project. This result contains the detail information related to the rest charts in this visualisation.
